# 抖音爆款选题推荐 Skill

## 快速概览

这个 Skill 做四件事：
1. **记住你的账号风格** — 保存在本地 `~/.douyin_prefs.json`，下次打开直接用
2. **抓取实时热榜** — 自动拉取抖音热榜 TOP30
3. **生成匹配选题** — 结合你的账号定位，分析并生成带理由的选题清单
4. **导出精美 Word 文档** — 自动生成 `.docx` 文件，带颜色分区、卡片排版，保存到桌面

无需任何 API Key。数据完全在用户本地，不上传任何内容。

---

## 依赖安装（首次使用提示）

首次运行前，检查依赖是否已安装：

```bash
python3 -c "import requests, bs4, docx" 2>/dev/null || pip3 install requests beautifulsoup4 lxml python-docx
```

如果安装失败，提示用户运行：
```bash
pip3 install -r <skill_dir>/scripts/requirements.txt
```

## 主工作流

### Step 1 — 读取用户偏好

```bash
python <skill_dir>/scripts/manage_prefs.py load
```

- **返回 `"exists": false`** → 进入「首次设置」流程
- **返回已有偏好** → 跳到 Step 3（抓取热榜）

---

### 首次设置 / 更新偏好

当用户第一次使用，或说"更新账号偏好"/"换赛道了"/"修改风格"时触发。

**A. 询问用户以下信息**（用中文友好地提问，不要像填表单）：

1. 抖音账号链接（可选，如有则自动分析）
2. 账号定位 / 垂直赛道（如：美食探店、职场干货、知识科普、生活 Vlog、母婴育儿、科技数码等）
3. 目标受众（如：25-35岁职场人、学生、宝妈、数码爱好者等）
4. 内容调性（如：轻松幽默、专业干货、温暖治愈、犀利评测）
5. 常用话题标签（可选，如 #职场# #探店# #副业#）

**B. 如果用户提供了账号链接，尝试自动分析**：

```bash
python <skill_dir>/scripts/analyze_account.py "<account_url>"
```

读取返回的 `public_info`，结合用户的手动描述，综合理解账号风格。如果抓取失败（抖音 JS 渲染限制），完全依赖用户手动描述。

**C. 保存偏好**：

```bash
echo '{
  "account_url": "<url 或空字符串>",
  "niche": "<账号定位，例如：美食探店>",
  "target_audience": "<目标受众>",
  "tone": "<内容调性>",
  "keywords": ["<标签1>", "<标签2>"],
  "extra_notes": "<用户补充说明>"
}' | python <skill_dir>/scripts/manage_prefs.py save
```

保存后告诉用户："✓ 账号偏好已保存，以后直接打开就能获取个性化推荐。"

---

### Step 3 — 抓取实时热榜

```bash
python <skill_dir>/scripts/fetch_trending.py
```

脚本只抓取**抖音热榜**，取前 30 条，输出包含完整热度值的 JSON。

**如果抓取失败**（网络问题），提示用户检查网络连接，或手动提供今日热榜话题。

---

### Step 4 — 生成选题推荐

基于：
- 用户偏好：`niche`、`tone`、`target_audience`、`keywords`
- 当天热榜：`topics` 列表（抖音热榜 TOP30，含标题 + 热度）

**每条选题必须明确关联一个或多个抖音热榜话题**，在 `hot_topic` 字段写清楚蹭的是哪个热点。

生成 **8-10 条选题建议**，分三类：

| 类型 | 逻辑 | 占比 |
|------|------|------|
| 🔥 热点蹭榜 | 热榜话题与账号赛道直接相关，制作对应内容蹭热度 | 3-4条 |
| 🌱 趋势借势 | 热榜揭示的社会情绪/趋势，从账号垂直角度切入 | 2-3条 |
| 💡 原创高潜 | 不依赖热点，基于账号调性和受众痛点的常青选题 | 2-3条 |

**选题标题写法**：参考 `references/output_format.md` 中的标题写法原则，写出有吸引力、可直接使用的标题。

**匹配理由**：说清楚"为什么这个话题适合这个账号"——受众共鸣点、流量逻辑、差异化角度。

分析完成后，**不要直接在对话中输出文字清单**，而是进入 Step 5，生成 Word 文档。

---

### Step 5 — 导出 Word 文档

将所有数据组装为如下 JSON，通过 stdin 传给 `generate_report.py`：

```json
{
  "date": "<YYYY-MM-DD HH:MM>",
  "prefs": {
    "niche": "<账号定位>",
    "target_audience": "<目标受众>",
    "tone": "<内容调性>"
  },
  "topics": [
    {
      "title": "<热点话题>",
      "heat": <热度值>,
      "category": "抖音热榜",
      "source": "抖音"
    }
  ],
  "recommendations": [
    {
      "type": "hot",
      "title": "<视频标题，带《》>",
      "hot_topic": "<关联热点话题，无则留空>",
      "reason": "<匹配理由，1-2句>",
      "shooting_tips": "<拍摄/创作建议，1句>"
    }
  ],
  "daily_summary": "<一句话今日内容方向总结>"
}
```

`type` 字段固定为三个值：`hot`（热点蹭榜）、`trend`（趋势借势）、`original`（原创高潜）。

运行命令：

```bash
echo '<上方组装好的 JSON>' | python3 <skill_dir>/scripts/generate_report.py
```

脚本会把文档保存到用户桌面，文件名格式：`选题推荐_YYYY-MM-DD.docx`，并在 stderr 打印路径。

生成后告诉用户："`📄 今日选题推荐已导出到桌面：选题推荐_YYYY-MM-DD.docx`"，并简要列出今天最值得做的 3 条选题标题。

### 更新偏好（随时可触发）

用户说"更新偏好"/"换赛道"/"修改账号信息"时：

1. 先读取现有偏好并展示给用户
2. 询问要修改哪些字段
3. 合并更新后保存

如果用户想完全重置：
```bash
python <skill_dir>/scripts/manage_prefs.py delete
```

然后重新走首次设置流程。

## 常见边缘情况

**用户没有抖音账号 / 还没开始做**：
- 直接询问"你打算做什么类型的内容？目标观众是谁？"，跳过账号分析
- 保存偏好时 `account_url` 设为空字符串

**用户是多平台创作者（小红书 / B站 / 视频号）**：
- 热榜数据同样适用，在选题建议中可备注"此选题也适合 [其他平台]"

**热榜数据抓取全部失败**：
- 告知用户网络限制，提供手动访问链接
- 提示："你也可以把今日热搜话题粘贴给我，我来帮你匹配选题"

**用户要求某个特定话题的选题**：
- 直接基于该话题 + 账号偏好生成相关选题，不需要重新抓热榜
